## KRX 데이터 수집

### 캐시 구조
- `krx_data_fetcher.py`의 `krx_data_cache.parquet`에 전체 주차 저장
- 컬럼: `단축코드`, `종목명`, `금융투자`, `은행`, `개인`, `week`
- 3개 Agent가 동일 파일에서 필요한 컬럼만 꺼내 씀

### 멘토님 엑셀 병합 현황 (2026.6.4 기준)
- **엑셀 13주** (3.2~5.29): 1065~1130개 ETF, 완전한 데이터
- **KRX 정상 3주** (1.12, 2.9, 2.16): 평균 723개
- **KRX 빵꾸 6주** (1.5, 1.19, 1.26, 2.2, 2.23, 6.1): 세션 만료로 부족
- → 빵꾸 주차는 `patch_all_weeks()` 로 보완 필요

### 주요 함수
```python
from krx_data_fetcher import (
    load_cache_recent,    # 최근 N주 로드 (증권사: 9주, 은행: 25주)
```

### 세션 관련
- KRX 세션 1시간 만료 → `fetch_weekly_etf_data` 내 55분 선제 재로그인 구현
- 과도한 재시도 시 KRX 서버가 당일 IP 차단 → 익일에 풀림
- `patch_week()`: 특정 주차 빠진 ETF만 보완 (전체 재수집 불필요)

## 증권사 채널 Agent

### 분석 흐름
```
채널 수집 → LLM 마케팅 감지 → ETF 특정 → 비교군 매핑 → DiD → 리포트
```

### 핵심 로직
- **기준 컬럼**: `금융투자`
- **베이스라인**: 8주 이동평균
- **라플라스 α**: 10억원 (분모 = 절댓값평균 + 10억)
- **LP 노이즈**: z > 2.0 AND 부호 반전 → 개인 컬럼으로 전환
- **비교군**: TIGER 우선 + ACE/PLUS (최대 2개)
- **자동 매핑**: 네이버 금융 기초지수 조회 + 이름 유사도

### DiD 판정
| DiD | 판정 |
|-----|------|
| ≥ 1.0 | 🟢 마케팅 효과 강함 |
| ≥ 0.3 | 🟡 마케팅 효과 있음 |
| ≥ -0.3 | ⚪ 효과 불분명 |
| < -0.3 | 🔴 유의미한 효과 확인 어려움 |

## 은행 채널 Agent

### 증권사와 근본적 차이
- 은행 마케팅은 앱 내부(팝업/PB)라 외부 크롤링 불가
- → **역방향 추적**: DiD 상시 감지 → 이상 시 역추적

### 수집 채널 (9개)
- 유튜브 RSS: KB국민은행, 신한은행, 하나은행, 우리은행, 농협은행
- 블로그: NH농협(네이버RSS), 하나은행(자체블로그)
- 뉴스: 네이버API + 구글뉴스RSS (은행+ETF 키워드)

### 2단계 DiD 알고리즘

**1단계 — 시장효과 제거:**
```
DiD(t) = KODEX_은행변화율(t) - 경쟁사_은행변화율(t)
변화율 = (현재 - 8주평균) / (절댓값평균 + 10억)
```

**2단계 — 평소격차 제거 (Z-score):**
```
Z(t) = (DiD(t) - DiD_16주평균) / (DiD_16주σ + 0.01)
```

**Z-score 판정:**
| Z | 판정 | 의미 |
|---|------|------|
| ≥ 2.0 | 🟢 강한 이상 감지 | 은행 마케팅 거의 확실 |
| ≥ 1.0 | 🟡 이상 감지 | 역추적 권고 |
| -1.0 ~ 1.0 | ⚪ 정상 범위 | 평소 패턴 |
| < -1.0 | 🔴 경쟁사 우위 | 경쟁 마케팅 의심 |

**은행 채널 특성:** σ가 작아서 Z=1.0이 일반 통계 Z=2.0과 동일한 실질 의미

```
→ "금융투자" → "은행" 전체 치환
```
