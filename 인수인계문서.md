## 핵심 분석 로직

### 1. 채널 수집 (collector.py)
실제 작동 채널 4개:
- 삼성자산운용 이벤트 페이지 (목록 전체 스캔, event.do)
- 삼성증권 유튜브 (RSS)
- 삼성증권 블로그 네이버 (RSS)
- 네이버/구글 뉴스 (RSS + 직접 스크래핑)

구조적 한계 (코드 수정으로 해결 불가):
- Instagram (봇 탐지), 카카오톡 (구독자 전용), 구글 트렌드 (공식 API 없음)
- KRX 거래실적 (SPA → 엑셀 업로드로 대체)

날짜 필터: 선택한 시트 날짜 범위로 RSS 항목 필터링

### 2. 마케팅 감지 (analyzer.py + app.py)
- Anthropic API 있으면 LLM으로 감지
- 없으면 엑셀의 전체 KODEX ETF 이름(236개)을 키워드로 스캔
- 감지 근거(채널·제목·링크·마케팅 분류 근거) 표시

### 3. 베이스라인
- 직전 4주 이동평균
- 현재 주 이전 시트만 사용 (시간 순서 엄수)

### 4. LP 노이즈 감지
- z > 2.0 **AND** 부호 반전 둘 다 해당할 때만 LP 의심
- LP 의심 시 금융투자 → 개인 기준으로 전환
- 비교군도 동일 지표로 전환 (공정 비교)
- 주의: 금리인하 등 장세 전반 전환 시 오탐 가능 (앱에 안내 표시됨)

### 5. DiD 계산
```
정규화 절대 변화 = (현재값 - 4주평균) / 4주절댓값평균

DiD = KODEX 초과분 - [(TIGER 초과분 + 타사ETF 초과분) / 2]
```

왜 이 공식: 비율법은 음수↔양수 전환 시 -588% 같은 폭발 발생 → 분모를 절댓값평균으로 대체

### 6. DiD 판정 기준
| DiD 값 | 판정 |
|--------|------|
| ≥ 1.0 | 🟢 마케팅 효과 강함 |
| ≥ 0.3 | 🟡 마케팅 효과 있음 |
| ≥ -0.3 | ⚪ 효과 불분명 |
| < -0.3 | 🔴 유의미한 효과 확인 어려움 |
| 비교군 없음 | ⚫ DiD 불가 |

※ 1.0 = KODEX가 비교군 대비 "평소 변동 크기만큼" 초과 판매됨을 의미
※ 임계값은 이론적 설정값 — 데이터 축적 후 실증 재보정 필요

## 비교군 매핑

### 하드코딩 (4개)
- 069500 KODEX 200 → TIGER 200(102110), PLUS 200(152100)
- 229200 KODEX 코스닥150 → TIGER 코스닥150(232080)
- 091160 KODEX 반도체 → TIGER 반도체(091230)
- 498400 KODEX 200타겟위클리커버드콜 → TIGER 동종(0104N0)

### 자동 매핑 (나머지 232개)
- 이름 유사도 기반 탐색
- 레버리지·인버스·커버드콜 등 변형 태그 불일치 시 제외
- 우선순위: TIGER > ACE/PLUS > SOL > 나머지
- 운용사별 1개씩 최대 2개 (같은 운용사 중복 없음)
- 비교군 1개면 각주 표시 ("시장에 1종만 존재")

1. **DiD 판정 임계값 미검증** — 실증 보정 필요
