# Agent 개발 참고사항
## 증권사 Agent 시행착오 → 타 채널 적용 시 참고

### 공통 주의사항
- **컬럼명**: KRX캐시=`단축코드`, 멘토님엑셀=`종목코드` → 반드시 분기처리
- **Laplace smoothing**: α=10억 (분모 0 방지, 소형ETF 폭발 방지)
- **베이스라인**: 8주 이동평균 (4주는 너무 짧음, 이벤트 lag 효과 때문)
- **주차 레이블**: `M.D-M.D` 형식, _parse_week_label()로 파싱

### 채널별 기준 컬럼
| Agent | 기준 컬럼 | 대조군 기준 |
|-------|---------|------------|
| 증권사 | 금융투자 | TIGER/ACE 동일 ETF |
| 은행   | 은행     | TBD |
| 매스   | 개인     | TBD |

### LP 노이즈 감지 (증권사 기준)
- z > 2.0 AND 금융투자 vs 개인 부호 반전 → LP 노이즈 의심
- 은행/매스는 LP 노이즈 구조 다름 → 별도 검토 필요

### DiD 계산
- 정규화 변화율 = (현재 - 8주평균) / |8주평균 + α|
- DiD = KODEX 변화율 - 대조군 평균 변화율
- 판정: +50%p↑ 강함 / +20~50%p 있음 / ±20%p 불분명 / -20%p↓ 없음

### 은행 Agent 특이사항
- 이벤트 감지가 선행 불가 → 순매수 이상감지 후 역추적
- 거래 빈도 낮아 베이스라인 안정적 → 스파이크 신호 더 선명
- 수집: 유튜브RSS 5개 + 구글뉴스(은행+ETF 키워드)

### KRX 수집
- 세션 1시간 만료 → 55분마다 선제 재로그인
- KODEX 우선 234개 수집 후 나머지
- 부족한 주차는 patch_all_weeks()로 보완
