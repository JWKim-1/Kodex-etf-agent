## 핵심 분석 로직

### 1. 채널 수집 (collector.py)
실제 작동 채널 4개:
- **삼성자산운용 이벤트 페이지** — event.do 목록 전체 스캔 + 본문 크롤링 (이미지 alt 포함)
- **삼성증권 유튜브** — RSS
- **삼성증권 블로그 (네이버)** — RSS
- **네이버/구글 뉴스** — RSS + 네이버 직접 스크래핑

구조적 한계 (해결 불가): Instagram, 카카오톡, 구글 트렌드, KRX SPA

날짜 필터: 선택한 시트 날짜 범위로 RSS 항목 필터링

### 2. 마케팅 감지
- Anthropic API 있으면 LLM으로 감지 + marketing_type/marketing_reason 반환
- 없으면 엑셀의 전체 KODEX ETF 이름(236개)을 키워드로 스캔
- 감지 근거 채널별 표시 + 마케팅 분류 근거 명시
- 미감지 채널도 별도로 표시 (수집됐으나 ETF 언급 없음)

### 3. 베이스라인
- 직전 4주 이동평균 (= 주봉 4일 이평선과 동일 개념)
- 현재 주 이전 시트만 사용 (시간 순서 엄수)
- [한계] 4주가 최적인지 미검증

### 4. LP 노이즈 감지
조건: z > 2.0 **AND** 부호 반전 **둘 다** 해당 시 LP 의심
- [한계] z > 2.0 임계값은 통계적 관례(95% 신뢰구간), 실증 미검증
- 비교군(TIGER)도 같은 부호 반전이면 장세 전환으로 처리 (LP 아님)
- [한계] 비교군 1개만 체크, 금리인하 등 시장 전반 전환 시 오탐 가능성
- LP 의심 시 금융투자 → 개인 기준 전환, 비교군도 동일 지표 사용

### 5. DiD 계산
```
정규화 절대 변화 = (현재값 - 4주평균) / 4주절댓값평균

DiD = KODEX 초과분 - [(TIGER 초과분 + 타사ETF 초과분) / 2]
```
왜 이 공식: 비율법은 음수↔양수 전환 시 -588% 폭발 → 분모를 절댓값평균으로 대체
[한계] 분모로 절댓값평균 vs 표준편차 vs 최댓값 중 최적 선택 미검증

### 6. DiD 판정 기준
| DiD 값 | 판정 |
|--------|------|
| ≥ 1.0 | 🟢 마케팅 효과 강함 |
| ≥ 0.3 | 🟡 마케팅 효과 있음 |
| ≥ -0.3 | ⚪ 효과 불분명 |
| < -0.3 | 🔴 유의미한 효과 확인 어려움 |
| 비교군 없음 | ⚫ DiD 불가 |

[한계] 임계값(1.0/0.3/-0.3)은 이론적 설정값 — 실증 재보정 필요
단위: KODEX가 비교군 대비 "평소 변동 크기(4주절댓값평균)"의 몇 배 초과했는지

## 비교군 매핑

### 하드코딩 (4개)
- 069500 KODEX 200 → TIGER 200(102110), PLUS 200(152100)
- 229200 KODEX 코스닥150 → TIGER 코스닥150(232080)
- 091160 KODEX 반도체 → TIGER 반도체(091230)
- 498400 KODEX 200타겟위클리커버드콜 → TIGER 동종(0104N0)

### 자동 매핑 (나머지 232개)
- **네이버 금융에서 기초지수 실시간 조회** (`finance.naver.com/item/main.naver?code=XXXXXX`)
- 기초지수 일치 시 +50점 보너스 (이름 유사도만 쓰던 것에서 개선)
- 레버리지·인버스·커버드콜 등 변형 태그 불일치 제외
- 운용사별 1개씩 최대 2개 (TIGER > ACE/PLUS > SOL 우선순위)
- [한계] 액티브 ETF는 자체 지수라 완전 일치 어려움 → 부분 일치 fallback
- 비교군 1개면 "시장에 1종만 존재" 각주 표시

## 마케팅 패턴 (실제 이벤트 데이터 분석 결과)

삼성자산운용 이벤트 데이터(2025.1~2026.6) 분석 결과 6가지 패턴 확인:
1. 신규/변경 상품 출시 — 초기 AUM 확보
2. 핵심 상품 꾸준히 밀기 — 미국대표지수, 배당 등
3. 테마 이슈몰이 — AI, 반도체, 우주항공 등 핫한 때
4. **AUM·시장 마일스톤** — 순자산 100조/200조 돌파, KOSPI 5000 등
5. **계절성 이벤트** — 설날, 봄맞이, 크리스마스 등
6. **경쟁사 대응** — 타사 보수 인하 시 맞불 이벤트

→ 케이스 4·5·6은 뉴스 키워드 추가로 감지 가능 (미구현)
